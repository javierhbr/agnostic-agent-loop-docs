## Gates in Markdown

Gates are checklist items inside the agent's output template — not a separate service:

```markdown
## Gate 3 — Implementation Spec Exit

### Checklist
- [x] implement-spec.md covers all components in feature-spec
- [x] Component MCP `get_invariants()` called — zero violations
- [x] Component MCP `get_patterns()` called — only approved patterns used
- [ ] **FAIL**: Cross-domain side effect detected — Cart writing to Checkout's Redis
- [ ] Domain Owner review pending

### FAIL Details
**Invariant violated**: CART-INV-001
**Rule**: Cart must not write to storage namespaces owned by other components
**Proposed fix**: Checkout creates its own Redis namespace `checkout-sessions:*`
```

This is the full gate implementation. No TypeScript. No build step. No CI plugin.
