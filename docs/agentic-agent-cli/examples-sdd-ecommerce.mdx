Register these under category `sdd-ecommerce-payments` and **subcategory `examples`**:

```bash
agentic-agent prompts list --category sdd-ecommerce-payments/examples
agentic-agent prompts show sdd-ecommerce-payments/examples/analyst
```

### `sdd-ecommerce-payments/examples/setup`

```text
Load the project sample config (00-setup/agnostic-agent.yaml) and mirror it in your repo.
Goal: seed .agentic/ config + contexts before the workflow starts.
Checklist:
- Copy agnostic-agent.yaml → edit org/name/roots
- Ensure .agentic/spec + tasks dirs exist
- Run `agentic-agent start` to validate config
Expected files: .agentic/agnostic-agent.yaml, spec dirs present.
Done when: CLI `start` passes without errors.
```

### `sdd-ecommerce-payments/examples/initiative`

```text
Initiative definition (01-initiative/initiative.md).
Prompt: Capture problem, goals, success metric, non-goals, timeline for Subscription Billing.
Make sure success has a metric (merchant adoption %, churn/retention).
Exit when: initiative.md has problem statement, success metric, constraints, stakeholders.
Outputs: initiative.md with Problem, Goals, Non-Goals, Timeline, Stakeholders.
```

### `sdd-ecommerce-payments/examples/risk`

```text
Risk assessment (02-risk-assessment/risk-classification.md).
Decide workflow = Full because payments + PII + data consistency.
Require gates + ADR before code.
Exit when: risk-classification.md marks CRITICAL and selects Full workflow.
Outputs: risk-classification.md, Workflow: Full, Risk: CRITICAL, Rationale documented.
```

### `sdd-ecommerce-payments/examples/analyst`

```text
You are running the Analyst phase of the SDD Full workflow.
Job: one-question-at-a-time discovery for Subscription Billing (critical risk).
- Call Platform MCP: get_context_pack("Subscription Billing")
- Ask for evidence (support tickets, lost deals, metrics)
- List affected components + blocking ADRs
- Classify risk (Critical) with rationale
Exit gates: metric in problem statement, evidence cited, components named, ADR-001 card vault, workflow=Full.
Start interview now; ask first question about problem statement.
Output: 03-analyst/discovery.md with evidence + blocking ADR list.
Key artifacts produced:
- discovery.md (evidence: 43 tickets, 8 lost deals)
- Affected components: Platform API, Billing, Payments, Email, User, Analytics
- Blocking ADRs: ADR-001 card vault
- Risk classification: CRITICAL → Workflow Full
```

### `sdd-ecommerce-payments/examples/architect`

```text
You are the Architect for SBL-2025-Q1 (Subscription Billing).
Inputs: discovery.md.
Call Platform MCP + Component MCPs (Payments, User, Email).
Produce:
- feature-spec.md (WHAT, 5+ GWT ACs, rollback plan, NFRs)
- component-spec-*.md (Billing, Payments, Email, User)
- ADR-001 card vault (Stripe vs self-hosted)
Every section needs Source: line; block on ADRs; self-check 5 gates.
Outputs: 04-architect/feature-spec.md, component-spec-*.md, ADR-001-card-vault.md.
Key contents:
- feature-spec.md: 7 ACs (G/W/T), rollback plan, metrics, observability, flags
- component-spec-billing.md: state machine, events, DB tables, contracts
- component-spec-payments.md: Stripe vault option, idempotency, retries
- component-spec-email.md: events consumed, templates
- component-spec-user.md: “has_active_subscription” flag
- ADR-001: Stripe vault chosen vs self-hosted
```

### `sdd-ecommerce-payments/examples/dev-billing`

```text
Developer agent for Billing Service.
Inputs: feature-spec.md, component-spec-billing.md, ADR-001 resolved (Stripe).
Produce impl-spec-billing.md + tasks.yaml (8-10 tasks).
Cover: data model, state machine, edge cases (cancel mid-cycle, retries, price change, crash after charge), observability, rollout/rollback.
Stop if blocked_by non-empty.
Outputs: 05-developer/impl-spec-billing.md and 05-developer/tasks.yaml (8-10 tasks).
Tasks example (from tasks.yaml):
- Create subscriptions table (id, customer_id, plan_id, state, next_charge_at)
- Implement renewals job with retries/backoff
- Emit subscription.* events
- Add metrics: charge_success_rate, charge_latency_p95
- Feature flag + rollback path
```

### `sdd-ecommerce-payments/examples/dev-payments`

```text
Developer agent for Payments Service (recurring charges).
Inputs: feature-spec.md, component-spec-payments.md, ADR-001 resolved.
Produce impl-spec-payments.md + tasks.yaml.
Cover: Stripe SDK integration, idempotency, retries/timeouts, card expiry, double-charge prevention, logging/metrics, rollout.
Outputs: impl-spec-payments.md + tasks.yaml.
Edge cases to cover:
- Stripe timeout → retry with idempotency key
- Card expired → notify via Email Service
- Gateway 500 → exponential backoff, alert
- Double-charge prevention → idempotency + dedupe store
```

### `sdd-ecommerce-payments/examples/verifier`

```text
Verifier phase before merge.
Inputs: feature-spec + component-specs + impl-specs + PR.
For every AC: gather evidence (test name+line, logs, metrics/trace).
Checks: lint, coverage ≥80%, PII/payment changes require human approval.
Outputs: verify.md; block on any failed/untestable AC.
Evidence examples:
- go test ./... output (lines)
- curl /health or subscription endpoints
- Logs for subscription.created/renewed
- Metrics: charge_success_rate, latency p95
```

### `sdd-ecommerce-payments/examples/gate-checks`

```text
Run gate 1-5 checks on all specs (see 07-gate-checks/gate-check-output.md).
Block if any gate fails; list remediation.
Deliverable: updated gate-check-output.md with pass/fail + fixes.
Gate examples:
- Gate 1 Context: initiative + risk present
- Gate 2 Design: feature-spec complete, ADR resolved
- Gate 3 Plan: tasks.yaml present per service
- Gate 4 Build: tests + lint pass
- Gate 5 Verify: verify.md with evidence
```
