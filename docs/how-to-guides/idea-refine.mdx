## The pipeline

```text
Vague input
  -> product-wizard (discovery + PRD)
  -> dev-plans (phases + checkpoints)
  -> atdd (test scenarios from ACs)
  -> code-simplification (remove vague/over-complex language)
```

## Quick prompt templates

```text title="Start from a vague idea"
Use the product-wizard skill to create a PRD for [idea].

Problem: [what problem?]
Users: [who needs this?]
Current state: [how they do it now?]
Success: [measurable outcome]
Constraints: [timeline/budget/tech limits]
Out of scope: [explicit exclusions]
```

```text title="Turn a PRD into an implementation plan"
Use the dev-plans skill to break this PRD into phases with checkpoints.

[paste PRD or reference file location]
```

```text title="Make acceptance criteria testable"
Use the atdd skill to show test scenarios for these acceptance criteria:

- [Criterion 1]
- [Criterion 2]
```

```text title="Simplify a complex requirement"
Use the code-simplification skill to rewrite this requirement in simpler,
testable language and call out ambiguities:

"[paste requirement]"
```

## If you want these inside `agentic-agent prompts`

This repo keeps the examples as documentation. If you want them discoverable from the CLI, a practical packaging is:

1. Add a prompt category (for example: `idea-refine`).
2. Add four prompt entries (for example: `vague-idea`, `requirements-list`, `existing-logic`, `refactor-legacy`).
3. Point each entry to the copy/paste “Initial prompt” blocks above.

That gives you a stable command surface like:

```bash
agentic-agent prompts list --category idea-refine
agentic-agent prompts show idea-refine/vague-idea
```
