# Software built as a *knowledge system*

A practical guide to **Spec-Driven Development** with two MCP servers, four Markdown agents, and zero overhead services.

<CalloutBox type="amber" mono>
  "Nothing is implemented without a validated spec backed by governed context." — Core Principle, v3.0
</CalloutBox>

## Problem Statement

- **Speed vs. correctness:** AI agents can write code in seconds, but they invent context when specs are weak.
- **Tribal knowledge does not scale:** contracts, invariants, and architecture decisions live in Slack and in people’s heads.
- **Process overhead kills velocity:** heavyweight validator services get abandoned or bypassed.

<CalloutBox type="navy" label="The answer">
  Two MCP servers serve governed knowledge. Four Markdown agents carry the process. Gates live as checklists in templates, not in extra TypeScript services.
</CalloutBox>
