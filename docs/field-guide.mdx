## Start With Your Scenario

### I found a bug (minutes to hours)

```bash
agentic-agent task create "Fix: <short description>"
agentic-agent task claim <task-id>
# ... code + tests ...
agentic-agent task complete <task-id>
```

Next: [/cli-cheatsheet](/cli-cheatsheet) (commands + gotchas), [/small-projects](/small-projects) (tiny workflow).

### I'm building a feature (1-2 weeks, single package)

```bash
# 1) write a short requirements.md (problem + acceptance criteria)
agentic-agent openspec init "Feature Name" --from requirements.md

# 2) work task-by-task
agentic-agent task claim <task-id>
# ... code + tests ...
agentic-agent task complete <task-id>

# 3) finish
agentic-agent openspec complete <change-id>
agentic-agent openspec archive <change-id>
```

Next: [/small-projects](/small-projects) (how-to + templates), [/cli-cheatsheet](/cli-cheatsheet) (one-page reference).

### I'm in a monorepo (multiple packages, parallel teams)

```bash
# root config enables multi-path spec discovery
# specs live near code (packages/*/specs)
agentic-agent openspec init "Feature Name" --from packages/<pkg>/specs/<spec>.md
```

Next: [/monorepo-openspec](/monorepo-openspec) (how-to), [/workflow-diagrams](/workflow-diagrams) (how discovery works).

### This is high-risk (payments/auth/PII/regulatory)

```bash
agentic-agent sdd start "Initiative Name" --risk critical
agentic-agent sdd workflow show <initiative-id>
agentic-agent sdd gate-check <spec-id>
```

Next: [/usage-levels](/usage-levels) (when to use which level), [/workflow-diagrams](/workflow-diagrams) (gate flow).
