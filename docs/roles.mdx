## Decision Authority Matrix

<CalloutBox type="amber" label="Key principle">
  When two roles disagree, the role whose **repo** owns the artifact has final authority. Platform Architect owns the constitution — not the Domain Owner. Domain Owner owns component invariants — not the Platform Architect.
</CalloutBox>

| Decision type | Authority |
|---|---|
| Platform-wide policy change | Platform Architect |
| New cross-cutting contract | Integration Owner + Platform Architect |
| Component invariant update | Domain Owner |
| Risk level classification | Platform Architect |
| Agent workflow modification | Platform Architect |
| Local ADR (component scope) | Component Team + Domain Owner |
| Global ADR (platform scope) | Platform Architect |
| Implementation approach (in-spec) | Component Team |
| Implementation approach (out-of-spec) | Domain Owner must approve first |

</div>

## Approval Gates by Role

When an agent produces output, it goes through human review at specific points:

| Agent output | Reviewer | Gate |
|---|---|---|
| analyst.md — initiative brief | Product Manager | Gate 1 |
| feature-spec.md — architecture | Platform Architect | Gate 2 |
| Contract changes in feature-spec | Integration Owner | Gate 2 |
| implement-spec.md — component plan | Domain Owner | Gate 3 |
| verify.md — verification report | Domain Owner (spot-check) | Gate 5 |
| Critical/High risk — final sign-off | Platform Architect | Gate 5 |
