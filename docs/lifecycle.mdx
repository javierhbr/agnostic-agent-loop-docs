## Workflow Selection

| Risk Level | Trigger | Workflow | Agents |
|---|---|---|---|
| **Low** | Single-component, no contract changes | Quick | Developer → Verifier |
| **Medium** | Multi-component or contract additions | Standard | Architect → Developer(s) → Verifier |
| **High / Critical** | Cross-cutting, new contracts, or `guaripolo` flag | Full | Analyst → Architect → Developer(s) → Verifier |
| **Any** | Production bug, revert needed | Hotfix | Developer → Verifier (expedited) |

</div>

<FlowStep
  role="Product Manager"
  roleAccent="pm"
  roleRepo="Platform Repo"
  title="Initiative Brief"
  desc="PM creates initiative in platform repo. Analyst agent (if full workflow) calls Platform MCP to load constitution and workflow template. Produces analyst.md with scope, affected components, and risk classification."

<FlowStep
  role="Platform Architect"
  roleAccent="arch"
  roleRepo="Specs Repo"
  title="Feature Specification"
  desc="Architect agent calls Platform MCP get_workflow(), get_contracts() for all affected components. Produces feature-spec.md: component changes, new contracts, ADRs. Architect reviews and approves Gate 2."

<FlowStep
  role="Component Teams"
  roleAccent="eng"
  roleRepo="System Repo"
  title="Implementation (parallel fan-out)"
  desc="Developer agent per component. Each calls Component MCP get_invariants(), get_patterns() before writing implement-spec.md. All developers run in parallel. Gate 3 per component."
/>

<FlowStep
  role="Domain Owners"
  roleAccent="comp"
  roleRepo="Specs + System Repo"
  title="Domain Review"
  desc="Domain Owner reviews implement-spec.md. Checks Gate 3 checklist: invariants respected, patterns followed, no cross-domain side effects. Signs off before implementation begins."
