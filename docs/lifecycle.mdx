## Workflow Selection

| Workflow | Risk | Agent sequence | Summary |
|---|---|---|---|
| **Quick** | low · bug_fix | Developer → Verifier | Fastest path, no high-level spec |
| **Standard** | medium · new feature | Architect → Developer → Verifier | No dedicated Analyst |
| **Full** | high · critical | Analyst → Architect → Developer → Verifier | Guaripolo, schema migrations, live trading |

## Architect Fan-out Rule

After `feature-spec.md` is approved, run one Developer agent per affected component in parallel. Verifier waits until all component tracks finish.
