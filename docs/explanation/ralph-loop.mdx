## CLI prompt catalog (subcategory: `examples`)

Register these slugs in your prompt catalog under the category `ralph-loop` and **subcategory `examples`**. The CLI will then surface them with:

```bash
agentic-agent prompts list --category ralph-loop/examples
agentic-agent prompts show ralph-loop/examples/start
```

### `ralph-loop/examples/start` (kick off the loop)

```text
/ralph-loop

I’m using the ralph-loop skill to iteratively complete this task.
- Read the spec & acceptance criteria.
- Write code, run tests, fix failures.
- Re-run until every criterion is met.
- Stop only when all checks pass and evidence is shown.
```

### `ralph-loop/examples/acceptance-criteria`

```yaml
# Good (specific, testable)
acceptance:
  - "go test ./... passes"
  - "GET /health returns 200"
  - "golangci-lint run shows 0 errors"
  - "Response includes version and uptime"

# Bad (vague)
acceptance:
  - "Code works well"
  - "Looks good"
  - "Ready for production"
```

### `ralph-loop/examples/verification-commands`

```bash
go test ./... -v
curl http://localhost:8080/health
golangci-lint run
go build ./cmd/...
```

### `ralph-loop/examples/troubleshooting`

```text
If AI skips verification → ask it to paste command output.
If loop stalls (no progress 3 iterations) → check service is running / ports / env.
If max iterations hit → split scope or relax unrealistic criteria.
```
