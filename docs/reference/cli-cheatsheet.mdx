## Quick Commands

### Tasks (simplest loop)

```bash
agentic-agent task create "Your task"
agentic-agent task list
agentic-agent task claim <task-id>
agentic-agent task complete <task-id>
```

### OpenSpec (lightweight spec + decomposition)

```bash
agentic-agent openspec init "Name" --from requirements.md
agentic-agent openspec list
agentic-agent openspec show <change-id>
agentic-agent openspec status <change-id>
agentic-agent openspec check <change-id>
agentic-agent openspec complete <change-id>
agentic-agent openspec archive <change-id>
```

### SDD (high-risk only)

```bash
agentic-agent sdd start "Name" --risk critical
agentic-agent sdd workflow show <initiative-id>
agentic-agent sdd gate-check <spec-id>
agentic-agent sdd adr create --title "Decision"
agentic-agent sdd adr resolve <adr-id>
```

### Full SDD 5-Phase (Platform + Multi-Team)

```bash
# Assess
agentic-agent task list

# Specify
agentic-agent openspec init "Feature" --from requirements.md

# Plan + Deliver
agentic-agent task claim <id>
agentic-agent validate
agentic-agent task complete <id>
```

**Core Boundary Rule:** Platform-side uses BMAD + OpenSpec + Speckit; component repos use OpenSpec ONLY.

## Workflow Comparison (Fast)

| Workflow | Use for | Typical timeline | Gates | Setup |
|---|---|---:|---|---:|
| `task` | Bugs, hotfixes, tiny refactors | hours to 1 day | none | 5 min |
| `openspec` | Features with acceptance criteria | 1-2 weeks | optional / lightweight | 10 min |
| `sdd` | High/critical risk changes | 3+ weeks | mandatory | 1 hour |

## Project Structure

### Single package

```text
myproject/
├── agnostic-agent.yaml
├── requirements.md
├── src/
└── .agentic/
    ├── openspec/changes/
    └── tasks/
```

### Monorepo

```text
mymonorepo/
├── agnostic-agent.yaml
├── specs/                 (optional shared specs)
├── packages/
│   └── <pkg>/
│       ├── specs/         (specs live with code)
│       └── src/
└── .agentic/
    ├── openspec/changes/  (central change tracking)
    └── tasks/
```

## Config Templates

### Minimal (single package)

```yaml title="agnostic-agent.yaml (single package)"
project:
  name: MyProject
  version: 0.1.0
agents:
  defaults:
    model: claude-3-5-sonnet-20241022
paths:
  openSpecDir: .agentic/openspec/changes
```

### Monorepo (multi-path spec discovery)

```yaml title="agnostic-agent.yaml (monorepo)"
project:
  name: MyMonorepo
  version: 0.1.0
agents:
  defaults:
    model: claude-3-5-sonnet-20241022
paths:
  openSpecDir: .agentic/openspec/changes
  specDirs:
    - packages/*/specs
    - apps/*/specs
    - specs
```

## Implementation Workflow (Plan -> Build -> Verify)

```text
Day 1: Plan
  write spec -> openspec init -> proposal.md + tasks.md

Days 2-N: Build
  claim task -> code + tests -> complete task

Final: Verify and close
  openspec status -> verify ACs -> openspec complete -> archive
```

## Prompt Shortcuts (When You Use an AI Agent)

For the full prompt library, see [/prompting-guide](/prompting-guide).

1. Spec review: "Break this spec into concrete tasks per acceptance criterion."
2. Package-specific: "Design + code sketch + files + tests for this task in `packages/<pkg>`."
3. Integration: "What does the consumer need to do? contracts + error handling."
4. Testing: "Test structure + mocks for these acceptance criteria."
5. Verification: "Show pass/fail evidence for each acceptance criterion."
6. Dependency analysis: "Implementation order + rollout strategy + mock plan."
7. Documentation: "Generate developer docs: overview, API, lifecycle, errors, testing, troubleshooting."

## Real Examples (Copy/Paste)

### Quick bug fix

```bash
agentic-agent task create "Fix login timeout"

```

### Small feature (single package)

```bash
cat > requirements.md << 'EOF'
# Product Search

## What
Add search + filters to product page.

- [ ] Search box appears
- [ ] Results filter in < 500ms
- [ ] Mobile responsive

agentic-agent openspec init "Product Search" --from requirements.md
```

### Monorepo (package-local specs)

```bash
mkdir -p packages/auth-service/specs

# OAuth Provider
## Problem
...
## Acceptance Criteria
- [ ] ...
EOF

agentic-agent openspec init "OAuth Provider" --from packages/auth-service/specs/oauth.md
```

## Decision Trees

### Should I use OpenSpec?

```text
Is this a bug fix?
├─ YES → Just use: agentic-agent task create
└─ NO → Continue...

Will it take > 1 week?
├─ NO → Use: agentic-agent task create
└─ YES → Continue...

Do you have clear acceptance criteria?
├─ NO → Write spec first, then: agentic-agent openspec init
└─ YES → agentic-agent openspec init --from requirements.md
```

### Single Package or Monorepo?

```text
Single package
├─ Use: /small-projects workflow
└─ Config: minimal agnostic-agent.yaml

Monorepo (multiple packages)
├─ Use: /monorepo-openspec workflow
└─ Config: multi-path specDirs
```

### When to Upgrade Workflows

```text
Current: Using tasks only
├─ Scale up if: > 2 week feature
└─ Next: agentic-agent openspec

Current: Using OpenSpec
├─ Scale up if: High risk (payment/auth/PII)
└─ Next: 5-Phase SDD (Platform > Assess > Specify > Plan > Deliver)

Current: Using 5-Phase SDD
├─ Add gates if: Regulatory/critical
└─ Command: agentic-agent validate
```

## Common Gotchas

| Problem | Fix |
|---|---|
| "Where do specs go?" | Single package: next to `requirements.md`; monorepo: `packages/*/specs/` |
| "Can I use specs and tasks?" | Yes: specs for planning, tasks for tracking |
| "What if the spec changes?" | Update the file and regenerate tasks as needed |
| "How do I integrate specs?" | Use `--from` flag: `openspec init --from other-spec.md` |
| "Can teams work in parallel?" | Yes, use monorepo workflow + multi-path config |
| "Do I need gates?" | Use gates for high-risk only; keep low-risk work fast |
| "Can I skip OpenSpec?" | Yes, use `task` commands directly |

## TL;DR Matrix

| Scenario | Use | Command |
|----------|-----|---------|
| Bug fix | Task | `task create "fix: X"` |
| Small feature | OpenSpec | `openspec init --from spec.md` |
| Monorepo | Multi-path + OpenSpec | See [/monorepo-openspec](/monorepo-openspec) |
| Critical feature | 5-Phase SDD | `openspec init + task claim` |
| Parallel teams | Monorepo + worktrees | `agentic-agent task claim` per team |
| Final verification | Hard gate | `openspec complete` (after tests pass) |
